# Bedrock Editionの仕様制約・できないこと一覧

> MC Bedrock 1.21.0〜1.21.60 対象 | Microsoft Learn + 実地検証 | 最終確認: 2026-02-28

## アイテムの制約

| 制約 | 詳細 |
|------|------|
| 3Dモデル | アイテム単体にカスタム3Dモデルは適用不可。attachable（装備時）のみ可能 |
| スタック数上限 | 最大64。64を超えるスタックは不可 |
| 独自エンチャント | アドオンでは新しいエンチャント種別の追加は不可（Script API除く） |
| 耐久値バー | minecraft:durability使用時のみ表示。カスタムバーは不可 |
| 左手装備 | 盾以外のカスタムアイテムは左手に装備不可 |
| ツール種別 | tool tier（木/石/鉄/ダイヤ/ネザライト）のカスタム定義は限定的 |

## ブロックの制約

| 制約 | 詳細 |
|------|------|
| ブロック数上限 | 1ワールドあたりカスタムブロックは約65,000種まで |
| ジオメトリサイズ | 最大 30×30×30 ピクセル（約2ブロック相当） |
| 透明度 | `alpha_test`か`blend`で透過可能だが、描画順序の問題あり |
| 水没対応 | `minecraft:unwalkable` 等で個別対応が必要 |
| ブロックエンティティ | カスタムブロックエンティティ（チェストのような）は不可。Script APIが必要 |
| レッドストーン | カスタムブロックのレッドストーン信号の入出力は非対応 |
| ピストン | カスタムブロックのピストン移動挙動のカスタマイズは限定的 |

## エンティティの制約

| 制約 | 詳細 |
|------|------|
| メシュ変形 | ランタイムでのメッシュ変形は不可。アニメーションで対応 |
| カスタムAI | 完全にカスタムなAIロジックは不可。既存behavior組み合わせのみ（Script API除く） |
| プレイヤーモデル変更 | プレイヤーのモデル自体の変更は不可。スキン変更のみ |
| ダメージタイプ | カスタムダメージタイプの追加は不可 |
| 複雑な当たり判定 | collision_boxは矩形のみ。円形や多角形は不可 |

## テクスチャの制約

| 制約 | 詳細 |
|------|------|
| 形式 | PNGのみ（JPG, GIF等は不可） |
| サイズ | 2の累乗推奨（16, 32, 64等）。非2累乗は描画問題の原因 |
| アニメーション | flipbook_textures.jsonで対応。動画テクスチャは不可 |
| 解像度上限 | 実質的に512×512程度が上限（パフォーマンス影響） |

## レシピの制約

| 制約 | 詳細 |
|------|------|
| 動的レシピ | ランタイムでのレシピ追加/変更は不可（Script API除く） |
| NBTクラフト | NBTデータを条件にしたクラフトは不可 |
| カスタムクラフトUI | 作業台のUI自体の変更は不可 |

## 全般的な制約

| 制約 | 詳細 |
|------|------|
| Script API | 高度な機能には @minecraft/server 等が必要（別フェーズ） |
| Moddingとの違い | Java版のModとは根本的に異なる。JSONベースのデータドリブンが基本 |
| バニラ上書き | バニラのアイテム/ブロックの挙動の完全な上書きには制限がある |
| マルチプレイ | RPはクライアントサイド、BPはサーバーサイド。両方必要 |
| マーケットプレイス | マーケットプレイス公開にはMicrosoft審査が必要 |
| パック優先順位 | 複数パック適用時は優先順位による。識別子重複に注意 |

## 「できない」の代替案

| やりたいこと | 不可能な方法 | 代替案 |
|-------------|------------|--------|
| カスタムGUI | ブロックにカスタムUI | Script API + forms |
| 新次元 | 新しいディメンション追加 | Script API (experimental) |
| 新バイオーム | バイオーム追加 | データドリブンバイオームは限定対応 |
| カスタムレッドストーン | ブロックにRS信号 | Script API でイベント処理 |
| 動くブロック | ブロックアニメーション | ブロック＋エンティティの組み合わせ |
